# cdp-tunnel v3.0.8 操作手册

## 一、升级

```bash
# 全局升级
npm update -g cdp-tunnel

# 确认版本
cdp-tunnel --version
# 应显示 3.0.8
```

## 二、重启 Proxy

升级后必须重启 proxy 才能加载新代码。

```bash
# 停掉旧 proxy（找到进程并 kill）
# macOS:
lsof -iTCP:9221 -sTCP:LISTEN -t | xargs kill

# 启动新 proxy
cdp-tunnel start
# 或
node /usr/local/lib/node_modules/cdp-tunnel/server/proxy-server.js
```

启动后确认版本和端口：
```bash
curl http://localhost:9221/json/version
# 应包含 "cdp-tunnel/3.0.8"

# 确认端口池端口已启动
curl http://localhost:9231/json/version
# 应返回 JSON（端口池端口）
```

## 三、端口说明

| 端口 | 模式 | 用途 |
|------|------|------|
| 9221 | create（现有） | CDP 创建新页面，强制分组隔离 |
| 9222 | takeover（现有） | 接管用户已打开的页面 |
| 9220 | takeover（端口池） | 接管用户浏览器（新） |
| 9231-9239 | create（端口池） | 每个端口 = 一个独立隔离环境（新） |

### 端口池行为（对齐原生 Chrome）

- **多客户端共享**：同一个端口可以有多个客户端同时连接，共享 tab
- **断开不清理**：客户端断开后 tab 仍存活，客户端需要自己用 `Target.closeTarget` 关闭
- **端口隔离**：不同端口的 client 互不可见对方的 tab
- **`/json/list` 天然隔离**：每个端口只返回自己的 tab

## 四、现有功能验证（9221/9222 不变）

### 4.1 基本连接

```bash
# Playwright 连接 9221（create 模式）
const browser = await chromium.connectOverCDP('http://localhost:9221');

# Puppeteer 连接 9221
const browser = await puppeteer.connect({ browserWSEndpoint: 'ws://localhost:9221/client' });
```

### 4.2 keyboard/mouse（v2.10.13 修复）

v2.10.13 修复了隔离 tab 上 keyboard/mouse 合成事件丢失的问题。
现在 `keyboard.type()`、`mouse.click()` 直接可用，无需客户端改动。

**代价**：每次 keyboard/mouse 操作时，隔离 tab 会短暂切到前台（Chrome 合成事件机制的硬限制）。

### 4.3 takeover 模式（9222）

```bash
# 连接 9222 接管用户当前浏览器
const browser = await chromium.connectOverCDP('http://localhost:9222');
```

## 五、端口池新功能验证（v3.0）

### 5.1 多会话隔离

```javascript
const { chromium } = require('playwright');

// 会话 A：连 9231
const browserA = await chromium.connectOverCDP('http://localhost:9231');
const pageA = await browserA.newPage();
await pageA.goto('https://example.com');
console.log('A tabs:', browserA.contexts()[0].pages().length);

// 会话 B：连 9232（和 A 完全隔离）
const browserB = await chromium.connectOverCDP('http://localhost:9232');
const pageB = await browserB.newPage();
await pageB.goto('https://example.org');
console.log('B tabs:', browserB.contexts()[0].pages().length);

// A 看不到 B 的 tab，B 看不到 A 的 tab
```

### 5.2 Network 事件监听

```javascript
const page = await browser.newPage();

// 监听网络请求
page.on('request', req => console.log('Request:', req.url()));
page.on('response', res => console.log('Response:', res.status(), res.url()));

await page.goto('https://example.com');
// 应看到所有网络请求的日志
```

### 5.3 截图（前台 + 后台）

```javascript
// 普通截图
const screenshot = await page.screenshot();
fs.writeFileSync('screenshot.png', screenshot);

// 后台流式截图（CDP screencast）
const cdpSession = await page.context().newCDPSession(page);
cdpSession.on('Page.screencastFrame', ({ data, metadata, sessionId }) => {
    console.log('收到帧:', data.length, 'bytes');
    cdpSession.send('Page.screencastFrameAck', { sessionId });
});
await cdpSession.send('Page.startScreencast', {
    format: 'jpeg', quality: 50,
    maxWidth: 800, maxHeight: 600
});
// ... 运行一段时间后
await cdpSession.send('Page.stopScreencast');
```

### 5.4 断开后 tab 存活（对齐原生 Chrome）

```javascript
const browser = await chromium.connectOverCDP('http://localhost:9231');
const page = await browser.newPage();
await page.goto('https://example.com');

// 断开连接
await browser.close();

// tab 仍然存活（对齐原生 Chrome 行为）
// 重新连接 9231，tab 还在
const browser2 = await chromium.connectOverCDP('http://localhost:9231');
// browser2 可以看到之前创建的 tab

// 如需关闭 tab，客户端自己调用：
// await page.close()  或  Target.closeTarget
```

## 六、配置

### 环境变量

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `PORT` | 9221 | 主 proxy 端口（create 模式） |
| `TAKEOVER_PORT` | PORT+1 | 主 takeover 端口 |
| `POOL_SIZE` | 9 | 端口池 create 端口数量（设 0 禁用） |
| `POOL_START` | 9231 | 端口池起始端口 |
| `POOL_TAKEOVER_PORT` | 9220 | 端口池 takeover 端口 |
| `LOG_LEVEL` | info | 日志级别（debug/info/warn） |

### 禁用端口池

```bash
POOL_SIZE=0 cdp-tunnel start
# 只启动 9221/9222，不启动端口池
```

```bash
LOG_LEVEL=debug cdp-tunnel start 2>&1 | tee proxy.log
```

## 八、已知限制

1. **keyboard/mouse 操作时 tab 短暂切到前台**——Chrome 合成事件机制的硬限制，无法绕过
2. **端口池的 `/json/version` 返回硬编码浏览器信息**——不影响功能，Playwright 照常工作
3. **端口池端口数固定**（默认 9231-9239）——可通过 `POOL_SIZE` 调整
