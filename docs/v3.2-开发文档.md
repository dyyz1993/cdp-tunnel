# cdp-tunnel v3.2 开发文档

## 概述

cdp-tunnel 是一个 Chrome 扩展 + Node.js proxy，让标准 CDP 客户端（Playwright/Puppeteer/agent-browser）通过 Chrome 扩展连接到用户已打开的浏览器，实现多会话隔离。

行为对齐原生 Chrome `--remote-debugging-port`——客户端无感接入。

## 架构

```
客户端（Playwright/Puppeteer/agent-browser）
    ↓ WebSocket
cdp-tunnel proxy（Node.js）
    ↓ WebSocket
Chrome 扩展（chrome.debugger API）
    ↓
用户的 Chrome 浏览器
```

## 端口布局

| 端口 | 模式 | 分组 | 用途 |
|------|------|------|------|
| 9220 | takeover | 不分组 | 接管用户已打开的 tab |
| 9221 | create | CDP-local-xxx | 扩展连接入口 + 默认 create 端口 |
| 9231 | create（端口池）| CDP-9231-xxx | 独立隔离环境 1 |
| 9232 | create（端口池）| CDP-9232-xxx | 独立隔离环境 2 |
| 9233 | create（端口池）| CDP-9233-xxx | 独立隔离环境 3 |
| 9234 | create（端口池）| CDP-9234-xxx | 独立隔离环境 4 |
| 9235 | create（端口池）| CDP-9235-xxx | 独立隔离环境 5 |
| 9236-9239 | create（端口池）| CDP-{端口}-xxx | 独立隔离环境 6-9 |

### 端口行为（对齐原生 Chrome）

| 行为 | 说明 |
|------|------|
| 多客户端共享 | 同一端口可以有多个客户端同时连接，共享 tab |
| 断开不清理 | 客户端断开后 tab 仍存活，需客户端自己 `Target.closeTarget` 关闭 |
| 端口隔离 | 不同端口的 client 互不可见对方的 tab |
| /json/list 天然隔离 | 每个端口只返回自己的 tab |
| 自动分组 | 端口池的 tab 自动归入 `CDP-{端口号}-xxx` 分组 |
| Browser.close | 关闭该端口的所有 tab |

## 快速开始

### 1. 安装

```bash
npm install -g cdp-tunnel
```

### 2. 安装 Chrome 扩展

1. 打开 Chrome → `chrome://extensions`
2. 开启"开发者模式"
3. 点击"加载已解压的扩展程序"
4. 选择 `node_modules/cdp-tunnel/extension-new` 目录
5. 扩展自动连接 `ws://localhost:9221/plugin`

### 3. 启动 proxy

```bash
cdp-tunnel start
```

或手动启动：
```bash
node node_modules/cdp-tunnel/server/proxy-server.js
```

### 4. 连接使用

#### Playwright

```javascript
const { chromium } = require('playwright');

// 连接端口池（独立隔离环境）
const browser = await chromium.connectOverCDP('http://localhost:9231');
const page = await browser.newPage();
await page.goto('https://www.example.com');
console.log(await page.title());
await page.screenshot({ path: 'screenshot.png' });
```

#### Puppeteer

```javascript
const puppeteer = require('puppeteer');

const browser = await puppeteer.connect({
  browserWSEndpoint: 'ws://localhost:9231/client'
});
const page = await browser.newPage();
await page.goto('https://www.example.com');
```

#### agent-browser

```bash
# 连端口池
agent-browser --session myapp --cdp http://localhost:9231 open "https://www.example.com"
agent-browser --session myapp --cdp http://localhost:9231 get url
agent-browser --session myapp --cdp http://localhost:9231 screenshot out.png
```

#### 裸 CDP（WebSocket）

```javascript
const WebSocket = require('ws');
const ws = new WebSocket('ws://localhost:9231/client');

ws.on('message', (data) => {
  const msg = JSON.parse(data);
  if (msg.id === 1) {
    console.log('createTarget result:', msg.result);
  }
});

ws.on('open', () => {
  ws.send(JSON.stringify({
    id: 1,
    method: 'Target.createTarget',
    params: { url: 'https://www.example.com' }
  }));
});
```

## 多会话隔离

每个端口池端口是独立的隔离环境——不同端口的 client 互不可见。

```javascript
// 会话 A：连 9231
const browserA = await chromium.connectOverCDP('http://localhost:9231');
const pageA = await browserA.newPage();
await pageA.goto('https://www.example.com');

// 会话 B：连 9232（和 A 完全隔离）
const browserB = await chromium.connectOverCDP('http://localhost:9232');
const pageB = await browserB.newPage();
await pageB.goto('https://www.example.org');

// A 看不到 B 的 tab，B 看不到 A 的 tab
```

## CDP 能力

### 已验证支持的 CDP Domain（17/17 对齐原生 Chrome）

| Domain | 支持的场景 |
|--------|---------|
| Target | createTarget / attachToTarget / closeTarget / getTargets |
| Page | navigate / captureScreenshot / startScreencast / addScriptToEvaluateOnNewDocument |
| Runtime | evaluate / consoleAPICalled |
| Input | insertText / dispatchKeyEvent / dispatchMouseEvent |
| Network | enable / requestWillBeSent / responseReceived |
| DOM | getDocument / querySelectorAll |
| Emulation | setDeviceMetricsOverride |
| Storage | Cookie（setCookie/getCookies/deleteCookies）/ localStorage |
| Fetch | enable / requestPaused / continueRequest |
| Performance | getMetrics |
| Tracing | start / end |
| Browser | close |

### 已知限制

| 限制 | 原因 | 影响 |
|------|------|------|
| Security domain 不支持 | chrome.debugger API 固有限制（返回 -32601）| 极少用，仅 HTTPS 证书错误处理 |
| keyboard/mouse 操作时 tab 短暂切前台 | Chrome 合成事件机制要求 visibility=visible | 对客户端透明（forward.js 自动处理） |

## 断开与重连

### 断开后 tab 存活（对齐原生 Chrome）

```javascript
const browser = await chromium.connectOverCDP('http://localhost:9231');
const page = await browser.newPage();
await page.goto('https://www.example.com');

// 断开（tab 不会消失）
await browser.close();
```

### 重连后看到之前的 tab

```javascript
// 重新连同一端口
const browser2 = await chromium.connectOverCDP('http://localhost:9231');
// 之前的 example.com tab 还在
const pages = browser2.contexts()[0].pages();
console.log(pages.length); // >= 1
```

### 关闭 tab

```javascript
// 方式 1：Playwright
await page.close();

// 方式 2：CDP 命令
await cdp.send('Target.closeTarget', { targetId: 'xxx' });

// 方式 3：关闭整个端口的所有 tab
await cdp.send('Browser.close');
```

## 截图

### 普通截图

```javascript
const screenshot = await page.screenshot();
fs.writeFileSync('out.png', screenshot);
```

### 后台流式截图（Screencast）

```javascript
const cdpSession = await page.context().newCDPSession(page);

cdpSession.on('Page.screencastFrame', ({ data, sessionId }) => {
  console.log('收到帧:', data.length, 'bytes');
  cdpSession.send('Page.screencastFrameAck', { sessionId });
});

await cdpSession.send('Page.startScreencast', {
  format: 'jpeg',
  quality: 50,
  maxWidth: 800,
  maxHeight: 600
});

// ... 运行一段时间

await cdpSession.send('Page.stopScreencast');
```

## Network 拦截

### 请求监听

```javascript
const cdpSession = await page.context().newCDPSession(page);
await cdpSession.send('Network.enable');

cdpSession.on('Network.requestWillBeSent', ({ request }) => {
  console.log('请求:', request.url);
});

cdpSession.on('Network.responseReceived', ({ response }) => {
  console.log('响应:', response.status, response.url);
});

await page.goto('https://www.example.com');
```

### 请求拦截（Fetch domain）

```javascript
await cdpSession.send('Fetch.enable', {
  patterns: [{ urlPattern: '*', requestStage: 'Request' }]
});

cdpSession.on('Fetch.requestPaused', async ({ requestId, request }) => {
  console.log('拦截到请求:', request.url);
  // 放行
  await cdpSession.send('Fetch.continueRequest', { requestId });
});
```

## 注入脚本

```javascript
const cdpSession = await page.context().newCDPSession(page);

// 在后续所有页面加载前注入
await cdpSession.send('Page.addScriptToEvaluateOnNewDocument', {
  source: 'window.__myFlag = true;'
});

await page.goto('https://www.example.com');
const result = await page.evaluate(() => window.__myFlag);
console.log(result); // true
```

## Takeover 模式（9220）

接管用户已打开的 tab（不创建新 tab，不分组）：

```javascript
const browser = await chromium.connectOverCDP('http://localhost:9220');
// 可以看到并操作用户当前打开的 tab
const pages = browser.contexts()[0].pages();
```

## 配置

### 环境变量

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `PORT` | 9221 | 主 proxy 端口（plugin + client） |
| `TAKEOVER_PORT` | 9220 | takeover 端口 |
| `POOL_START` | 9231 | 端口池起始端口 |
| `POOL_SIZE` | 9 | 端口池端口数量 |
| `POOL_TAKEOVER_PORT` | 9220 | 端口池 takeover（同 TAKEOVER_PORT）|
| `LOG_LEVEL` | info | 日志级别（debug/info/warn）|

### 禁用端口池

```bash
POOL_SIZE=0 cdp-tunnel start
```

### 版本检查

```bash

# 确认 "cdp-tunnel/x.y.z" 版本号
```

```bash

```

```bash

```

```bash
LOG_LEVEL=debug cdp-tunnel start 2>&1 | tee proxy.log
```

## 升级

```bash
# 1. 升级 npm 包
npm update -g cdp-tunnel

# 2. 重启 proxy
kill $(lsof -iTCP:9221 -sTCP:LISTEN -t)
cdp-tunnel start

# 3. reload 扩展
# chrome://extensions → CDP Bridge → 刷新按钮
```

## 版本兼容性

| 版本 | 端口 | 说明 |
|------|------|------|
| v2.x | 9221 create + 9222 takeover | 老架构（断开清理 tab）|
| v3.x | 9220 takeover + 9221 plugin + 9231-9239 create | 新架构（端口池，对齐 Chrome）|

v3.x 兼容 v2.x 的 9221/9222 端口（现有客户端无需改动）。
