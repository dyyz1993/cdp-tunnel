Why CDP Tunnel?

Zero Config

One command cdp-tunnel start auto-launches Chrome with the extension. No flags, no setup.

Playwright & Puppeteer

Drop-in compatible. Just connect to http://localhost:9221 — works with both frameworks.

Multi-Client Isolation

Multiple automation clients, each with isolated tab groups. No interference between sessions.

Your Real Browser

Keep your login state, cookies, and extensions. No fresh profiles — automate your actual browser.

Auto Recovery

--watchdog crash recovery and --auto-restart Chrome restart. Your automation keeps running.

6-Step Diagnose

cdp-tunnel diagnose finds and fixes common issues automatically. Six health checks in one command.

Quick Start

$ npm i -g cdp-tunnel

$ cdp-tunnel start              # Start proxy + launch Chrome
$ cdp-tunnel start --watchdog   # With crash recovery
$ cdp-tunnel start --auto-restart  # Auto-restart Chrome
$ cdp-tunnel status             # Check server & extension
$ cdp-tunnel diagnose           # 6-step health check
$ cdp-tunnel update             # Auto-update + restart
$ cdp-tunnel stop               # Stop everything

Code Examples

Playwright

const { chromium } = require('playwright');

// Connect to your real browser
const browser = await chromium.connectOverCDP(
  'http://localhost:9221'
);
const page = await browser.newPage();
await page.goto('https://example.com');

Puppeteer

const puppeteer = require('puppeteer-core');

// Same idea, different API
const browser = await puppeteer.connect({
  browserURL: 'http://localhost:9221'
});

// Use your existing profile
const pages = await browser.pages();
